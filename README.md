# 图像盲水印与版权保护系统 — 使用说明

## 一：快速开始

本系统已升级为**可视化网页端（Web UI）**，无需再通过命令行敲代码，所有操作均可在浏览器中点点鼠标完成！

### 🚀 最简单的启动方式（Windows 用户）
直接在项目文件夹中，双击运行 **`双击我运行系统.bat`** 文件。
* 脚本会自动检测环境、安装依赖，并自动在浏览器中打开系统网页。
* **注意**：运行期间会弹出一个黑色的命令行窗口，**请不要关闭它**！关闭它网页就会断开连接。

### 💻 手动启动方式（Mac / Linux ）
如果你不想用 `.bat` 脚本，或者使用的是 Mac/Linux 系统，请按以下步骤操作：

1. **打开终端**，进入项目根目录。
2. **创建并激活虚拟环境**：
   ```bash
   python -m venv venv
   # Windows:
   venv\Scripts\activate
   # Mac/Linux:
   source venv/bin/activate
   ```
3. **安装依赖**：
   ```bash
   pip install -r requirements.txt
   ```
4. **启动网页前端**：
   ```bash
   streamlit run app.py
   ```
   运行后，浏览器会自动打开 `http://localhost:8501`。

## 二：三大核心功能模块

系统左侧导航栏提供三个核心功能，完全在本地运行，**绝对安全，不会上传任何图片到云端**。

### 1. 📸 水印嵌入
**作用**：将你的专属黑白水印（如签名、Logo）隐形嵌入到彩色摄影原图中。
* **操作**：上传原图 -> 上传水印 -> 点击“打入隐形水印”。
* **效果**：生成的图片肉眼看不出任何区别，但内部已经打上了你的版权烙印。你可以放心地把这张图公开发布到网上。
* **防二次篡改**：系统内置特征检测，如果原图已经包含过盲水印，程序会精准识别并**拒绝二次打入**，防止他人用你的含水印图强行覆盖洗稿。

### 2. 🛡️ 快捷验证
**作用**：在网上看到一张可疑的盗图，怀疑是你的作品？直接用这个功能验证。
* **操作**：上传可疑盗图 -> 上传你的原始水印 -> 点击“盲验证版权”。
* **效果**：**不需要提供原图**，系统直接从可疑图片中盲提取出隐藏的水印轮廓，并与你的原始水印进行比对。
* **判定**：系统会计算 NC（相关系数），并直接在网页上告诉你“是否包含你的水印”。

### 3. 🔍 取证对比
**作用**：当你确认图片被盗用，需要向平台投诉或提交法庭证据时使用。
* **操作**：上传原图 -> 上传含水印图 -> 上传原始水印 -> 点击“开始版权取证”。
* **效果**：系统会生成一张**全证据链对比图**。图中会详细标出原图被修改的像素点、盲提取出的水印图案，以及逐像素比对的匹配率，形成无可辩驳的视觉证据。

## 三：判定逻辑与原理

### 1. 判定标准 (NC 相关系数)
系统提取水印后，会计算 **NC（零均值归一化交叉相关，ZNCC）**。
*(注：ZNCC 算法能有效消除图像亮度的影响，使得随机噪声的匹配度严格趋近于 0，完全匹配为 1)*

| NC 值 | 含义 | 判定 |
|-------|------|------|
| > 0.70 | 提取出的水印与原水印高度一致 | 包含你的水印 |
| < 0.30 | 提取出的图案接近纯随机噪声，或为不相干水印 | 不含此水印 |
| 0.30~0.70 | 介于噪声与匹配之间 | 可能有微量残留（图片可能被严重破坏） |

### 2. 水印原理
水印不会画在图上——它修改的是图片 Y 通道（亮度）的数学结构：
1. 把图片缩放到 512×512，分成 64×64 = 4096 个 8×8 小块。
2. 每个小块做 DCT 变换，取出两个中频系数 D(4,3) 和 D(3,4)。
3. 根据水印签名上对应位置的像素（黑/白），微调这两个系数的大小关系。
4. 逆变换回去，肉眼完全察觉不到。

提取时反过来：对图片做同样的 DCT 变换，比较每个小块系数的大小关系，盲恢复出 64×64 的水印图案。4096 个像素对的系数关系同时恰好匹配的概率为 (1/2)^4096 ≈ 10^-1233。

## 四：系统日志 (System Log)

为了方便排查问题，系统在运行过程中会自动在项目根目录下生成一个名为 `system.log` 的日志文件。
* **日志内容**：记录了程序的启动/结束时间、成功处理的图片信息、被跳过的异常文件警告，以及底层的报错堆栈信息。
* **滚动机制**：日志文件最大为 1MB，超过后会自动备份（最多保留 3 个历史备份），不会无限占用硬盘空间。
* **排错指南**：如果网页端提示“处理失败”，你可以点击网页上的“查看后台日志”折叠框，或者直接打开 `system.log` 查看详细的错误原因。

## 五：常见问题与系统局限性 (FAQ)

**Q: 如果有人从网上下载了我没有打水印的原图，然后打上了他自己的水印，我该怎么证明图片是我的？**

**A:** 这是一个经典的“盗贼抢注”问题（先发制人困境）。从纯技术的角度来说，盲水印系统本质上是一把**“隐形的锁”**。如果你自己不给自行车上锁（不打水印）就扔在街上，小偷推走后挂上了他自己的锁，系统检验时自然只能得出“这把锁是小偷的”的结论。

**水印技术是“事前防御”工具，而不是“事后追溯”工具。** 如果图片本身是纯净的，代码不可能凭空变出原作者的版权信息。

但在现实的版权纠纷中，原作者可以通过以下方式进行来证明版权：
1. **时间戳（Prior Art）**：你在微博、小红书、Pixiv 或个人网站上发布这张图片的时间，绝对早于小偷打完水印后发布的时间。互联网是有记忆的，发布时间就是铁证。
2. **源文件（终极杀招）**：原作者手里有相机的 **RAW 格式原片**，或者画师手里有带几百个图层的 **PSD 工程文件**。而小偷从网上扒下来的，只是一张被压缩过的、扁平的 JPG 图片。只要亮出工程文件，小偷瞬间哑口无言。
3. **分辨率与画质**：原作者手里有 4K 甚至 8K 的超高清无损原图。小偷扒下来的往往是平台压缩过的 1080P 甚至更糊的图。小偷就算把糊图打上水印，也变不出高清细节。
